  - Subject in the right panel contains HTML
  - Right now token overrides are manually returning the token in instances that we think the token is not available. What we could do is update the @replace-tokens.query.ts so that if a value cannot be replaced (i.e. it the token replace function returns empty string or null or maybe undefined) then in that instance we return the token name back to the full string we are replacing tokens in. So, I think the best way to handle that is to allow the caller of the replace tokens function to pass in an enum that represents how they want the token represented. Either {{TOKEN_NAME}} or <span>{{TOKEN_NAME}}<span> or '' (empty string). So if the enum is "token-name" then it's just the token name without replacement. If it's "token-name-highlighted" then it's the span you've seen with the colors etc. And if it's "empty-string" then it's an empty string. This would solve the problem more elegantly and lead to fewer edge cases where we need to fix a specific token because the context is different. The token replacement will just either work or it wont given the context we provide and if it doesn't then it's returned in the format we describe with the enum.
